# インフラドキュメント

PaperBuddyのインフラドキュメントは、考え方・構成を説明するドキュメントと、実際に作業する手順書を分けて管理する。

## 読み分け

| 目的 | 参照先 |
|---|---|
| AWS上の全体構成、公開範囲、リソース設計を理解する | [アプリケーション構成](application.md) |
| VPC、サブネット、通信経路、セキュリティ境界を確認する | [ネットワーク構成](network.md) |
| 開発環境をAWSへ完全初回デプロイまたは差分デプロイし、PDF取り込みと動作確認を行う | [AWSデプロイ手順](deployment-guide.md) |
| PaperBuddy関連AWSリソースを完全削除する | [AWS完全削除手順](deletion-guide.md) |

## 記載方針

- `*-guide.md`は作業者が上から順に実行する手順を書く。ß
- 構成ドキュメントには、リソースの役割、設計判断、制約、運用上の注意を書く。
- 同じコマンドを複数ファイルに重複して書かない。代表手順は手順書へ集約し、構成ドキュメントからリンクする。
- CloudFrontを唯一の公開入口とし、ALBとS3を直接公開しない前提は全インフラ文書で維持する。
