# チャット削除ユースケース仕様書

## 概要

認証済みユーザーが所有する、パスパラメータで指定されたチャットIDに紐づくDynamoDB項目を削除する。

## 入力

- `chat_id`: パスパラメータで指定されたチャットID
- `user_id`: 認証済みユーザーID

## 正常系

1. チャット本体を取得し、所有ユーザーが認証済みユーザーと一致することを確認する。
2. `CHAT#{chat_id}`をパーティションキーに持つ全項目を取得する。
3. チャット本体と全メッセージを削除する。

## 制約

- DynamoDBの`BatchWriteItem`上限に従い、25件単位で削除する。
- 複数バッチにまたがる削除は原子的ではない。途中で失敗した場合、一部項目が削除済みとなる可能性がある。

## 異常系

- DynamoDBのQueryまたはBatchWriteに失敗した場合は`ChatDeleteError`を送出する。
- 未処理項目が返された場合は`ChatDeleteError`を送出する。
- 対象チャットが存在しない、または所有ユーザーが一致しない場合は`ChatNotFoundError`を送出する。

## API

- `DELETE /api/chats/{chat_id}`
- 成功時: `204 No Content`
